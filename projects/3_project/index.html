[OBJECT] Creation

Background

When a prospective customer has agreed to enter a contract with [COMPANY], an [OBJECT] is created under the customer’s account in [SERVICE]. This kicks off an automated request to create the [OBJECT] via our backend services. Most requests are processed within 2-3 minutes; however, if the request is unsuccessful or has been left open for 3 hours, [SERVICE] will automatically mark the [OBJECT] record as Create Failed, which will alert the Support Admin that manual creation is required.

Automated [OBJECT] creations include creating the first Admin User(s) as part of the process, while manual [OBJECT] creations require the first Admin User(s) be created in a separate flow (Note: This will change upon GA of the new [SERVICE] tool). For Paid [OBJECT], Admin Users will receive their claim links upon creation; for Trial [OBJECT], they must be manually sent their claim links. This is to ensure Admin Users are properly oriented to the Trial product before they gain access.

When should you integrate?

Squads will need to integrate with [OBJECT] Creation if they have processes that must be completed when a new [OBJECT] is created.

Process

Automated

  • The customer agrees to enter a contract with [COMPANY]
    • For Paid, the [OBJECT] is transitioned to [STATE]
    • For Trial, the [OBJECT] is Approved
  • An [OBJECT] record is created in [SERVICE] (representing an [OBJECT] creation request)
  • [SERVICE] picks up the request and adds it to the queue
    • For Paid [OBJECT], requests are added immediately
    • For Trial [OBJECT], requests will only be added if the Trial Start Date is on or before the current date
  • Once the [OBJECT] is created, [SERVICE] will make a request to create the Admin Users
  • [SERVICE] will also send notifications to all services integrated into its lifecycle operations
  • [SERVICE] closes the request by marking the [OBJECT] as Paid, Trial, or Create Failed

Manual

  • The customer agrees to enter a contract with [COMPANY], falling into one of the following cases:
    • [REDACTED]
    • [REDACTED]
    • [REDACTED]
  • An email is sent to a TSE requesting manual [OBJECT] creation
  • The TSE creates the [OBJECT] via the [SERVICE] tool in the [COMPANY] client
  • Once the [OBJECT] is created, the TSE manually creates the Admin Users (Note: This will change upon GA of the new [SERVICE] tool)
  • Finally, the TSE notifies the [OBJECT] owner that the [OBJECT] has been created
